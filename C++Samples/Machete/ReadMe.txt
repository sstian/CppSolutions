environment: Microsoft Visual Studio Professional 2017,  EasyX 20170421(beta)

***游戏说明：***
    一个地方很经典的对战纸牌游戏，通过互相出牌或者攻击直至赢得本局决斗的胜利。为简化操作，将其设计成用鼠标进行操作。

***操作指南：***
    鼠标：
在主菜单界面、游戏模式选择界面=》左键点击进行选择，右键点击退出游戏/返回上一界面。
在游戏决斗场面=》左键点击选择要出的手牌，右键则手牌重置且不出牌。
（特别说明：在牌堆上点击鼠标左键会发起投降。）

***设计经历：***
    这个小游戏制作的想法在两年前就有了，但是由于当时水平有限，不知该如何下手。虽然自学了C、C++、Java、Android等编程方面的东西，仍然不能很好地设计出一个GUI（图形用户界面）。而用windows 的MFC编程实在是太麻烦了，由其是网络编程。不得不多，对于纯粹的面向对象的Java来说，只需要用到什么就调用什么，不必关心内部实现细节，实在是太方便不过了。但是，在本人看来，C++才是王道。直到在2017年“520”之际接触了easyX简易的C++图形库，实现一个图形界面易如反掌。当时有老师讲520表白程序的制作说明，一时间来了兴趣。第二天开始学习帮助文档和EasyX网站（http://www.easyx.cn/）的优秀例程，获益匪浅。在积累了一些相关知识和技能之后，并查阅了windows中的API之后，就开始着手准备了。
    但是由于时间有限，面临着大学本科最后的毕业季时节，明天（06-12）就开始终期答辩了，而我的毕设挂图还没有画，后续还有五六事项需要处理，只能到这里为止了。11天独自儿写了2700多行代码，还是头一次做这么多代码量的项目。有些没有实现的功能估计也没时间再花工夫投入设计了。不足三个月后就要去新的学校开启研究生的学习之旅。转眼人已快到中年，时间紧任务重，想有那时候轻松玩乐的心情不易，想像那时候单纯自由的生活亦不易。
    生活不易，行旅维艰，且行且珍惜。

***不足之处：****
1、主菜单界面：选项 功能未添加“游戏难度”、未显示用户信息（如名字）的内容。帮助 功能未绘制或加载完整的游戏设计文档和详细的游戏机制说明。
2、游戏模式界面：只实现了单机模式，还有双人、混战、联机模式未完善，估计需要网络编程、socket套接字等C++技能。但是，你见过五角星形的窗口界面吗？
3、游戏决斗前：未实现“无限剑制咏唱词”的语音与文字的加载出现；未进行游戏先后手的卡牌猜先（默认玩家为先手）。
4、游戏决斗中：鼠标灵敏度不够，右键PASS掉的实现要点几下才会有反应。
5、游戏决斗后：决斗完后直接跳转回菜单主界面，但是却保留上局游戏的参数。
6、游戏机制：未加入“四组同可作攻防6点”游戏规则；牌堆抽完实际是将用过的洗切重新置入牌堆，如此循环直至一方输掉游戏，程序改为若牌堆无牌可抽，则双方平局；CPU出牌太单调，它为先手时只会出手牌中最小的单张，不会攻击也没有组成顺子的思考方式，所以游戏结果可想而知：要么玩家胜利，要么双方平局。实则这是机器算法的问题。
7、游戏配置：未加入游戏配置INI文件，将用户数据写入文件中以便下次启动时直接加载并保留上次游戏参数。
8、游戏音效：对于按钮特效和选中效果以及卡牌出牌、战斗音效等并未设置，这需要搜集素材，根据场景添加。（爱给网_影视和游戏素材_免费下载：http://www.aigei.com/）
